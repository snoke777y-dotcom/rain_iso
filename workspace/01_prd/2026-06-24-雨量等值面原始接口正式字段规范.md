# 雨量等值面原始接口正式字段规范

## 1. 文档目标

本文用于把雨量等值面前端当前使用的两份原始接口样例，正式固化为字段级协议定义。

本文只覆盖：

- `5分钟分时` 原始接口
- `1小时累计` 原始接口
- 顶层结构
- 时间键格式
- 单站记录字段
- 前端整理与校验口径

本文不覆盖：

- 静态空间资产
- Worker 消息协议
- 成图算法

---

## 2. 协议来源

当前正式协议源为以下两份真实返回样例：

- `datas/01_raw/realtime_5m_response.txt`
- `datas/01_raw/realtime_1h_response.json.txt`

约定：

- 后续前端实现按这两份样例的字段结构冻结
- 若服务端字段发生变化，必须先更新本文档，再允许改动前端解析逻辑

---

## 3. 接口范围

系统当前消费两类原始接口：

| 产品类型 | 本地样例文件 | 时间步长 | 用途 |
| --- | --- | --- | --- |
| `5分钟分时` | `datas/01_raw/realtime_5m_response.txt` | `5分钟` | 生成分时等值面 |
| `1小时累计` | `datas/01_raw/realtime_1h_response.json.txt` | `60分钟` | 生成起报累计等值面 |

说明：

- 两类接口顶层结构相同
- 两类接口单站字段结构相同
- 差异只在时间步长和雨量语义

---

## 4. 顶层结构

### 4.1 字段集合

顶层字段集合固定为：

- `data`
- `msg`
- `code`

### 4.2 结构定义

```json
{
  "data": {
    "2026-06-24 13:55:00": [
      {
        "sysid": "FXYJ0523",
        "stcd": "1404CD8EFA",
        "stnm": "流村镇发电站村",
        "rvnm": "",
        "hnnm": "",
        "lgtd": 115.97749,
        "lttd": 40.119777,
        "stlc": "",
        "addvcd": "110114000000000",
        "addvnm": "昌平区",
        "adnm": "",
        "stlvl": "12",
        "admauth": "险村",
        "isOut": "",
        "drp": 0.2,
        "bscd": "",
        "bsnm": "",
        "area": "110100",
        "star": 0
      }
    ]
  },
  "msg": "操作成功",
  "code": "0"
}
```

### 4.3 顶层字段定义

| 字段 | 类型 | 是否必填 | 含义 | 当前口径 |
| --- | --- | --- | --- | --- |
| `code` | `string` | 是 | 接口状态码 | 当前必须等于 `"0"` |
| `msg` | `string` | 是 | 接口返回说明 | 当前样例为 `操作成功` |
| `data` | `object` | 是 | `时间字符串 -> 站点数组` 映射 | 不允许为空 |

## 5. 时间键定义

### 5.1 原始格式

`data` 的 key 使用以下原始格式：

```text
YYYY-MM-DD HH:mm:ss
```

示例：

- `2026-06-24 13:55:00`
- `2025-07-23 21:00:00`

### 5.2 前端整理规则

前端读取后统一执行：

1. 读取 `data` 的全部时间键
2. 将空格时间格式转为 `ISO 8601 +08:00`
3. 按时间升序排序
4. 作为内部帧序列的 `frameTime`

转换示例：

| 原始时间键 | 内部标准时间 |
| --- | --- |
| `2026-06-24 13:55:00` | `2026-06-24T13:55:00+08:00` |
| `2025-07-23 21:00:00` | `2025-07-23T21:00:00+08:00` |

### 5.3 时间步长约束

| 产品类型 | 校验规则 |
| --- | --- |
| `5分钟分时` | 相邻时间差必须为 `5分钟` |
| `1小时累计` | 相邻时间差必须为 `60分钟` |

说明：

- 时间键在原始返回中不保证天然有序
- 前端必须自己排序，不能依赖接口返回顺序

## 6. 单站记录字段定义

### 6.1 字段集合

单站记录字段集合按样例冻结为：

- `sysid`
- `stcd`
- `stnm`
- `rvnm`
- `hnnm`
- `lgtd`
- `lttd`
- `stlc`
- `addvcd`
- `addvnm`
- `adnm`
- `stlvl`
- `admauth`
- `isOut`
- `drp`
- `bscd`
- `bsnm`
- `area`
- `star`

### 6.2 字段表

| 字段 | 类型 | 是否必填 | 示例 | 说明 |
| --- | --- | --- | --- | --- |
| `sysid` | `string` | 是 | `FXYJ0523` | 系统内部站点标识 |
| `stcd` | `string` | 是 | `1404CD8EFA` | 站点编码，前端以此作为稳定主键 |
| `stnm` | `string` | 是 | `流村镇发电站村` | 站点名称 |
| `rvnm` | `string` | 是 | `洳河` 或 `""` | 河流名称，可为空串 |
| `hnnm` | `string` | 是 | `潮白河流域` 或 `""` | 流域名称，可为空串 |
| `lgtd` | `number` | 是 | `115.97749` | 经度 |
| `lttd` | `number` | 是 | `40.119777` | 纬度 |
| `stlc` | `string` | 是 | `""` | 站址描述，可为空串 |
| `addvcd` | `string` | 是 | `110114000000000` | 行政区划编码 |
| `addvnm` | `string` | 是 | `昌平区` | 行政区划名称 |
| `adnm` | `string` | 是 | `""` | 附加行政名称，可为空串 |
| `stlvl` | `string` | 是 | `"12"` | 站点级别，当前以字符串返回 |
| `admauth` | `string` | 是 | `险村` | 管理或来源口径字段 |
| `isOut` | `string` | 是 | `""` | 当前样例为字符串字段 |
| `drp` | `number` | 是 | `0.2` / `82.6` | 当前时间片对应雨量值 |
| `bscd` | `string` | 是 | `""` | 预留字段，可为空串 |
| `bsnm` | `string` | 是 | `""` | 预留字段，可为空串 |
| `area` | `string` | 是 | `110100` | 区域代码 |
| `star` | `number` | 是 | `0` | 当前样例为数值字段 |

### 6.3 前端当前直接依赖字段

前端当前计算直接依赖以下字段：

- `stcd`
- `stnm`
- `lgtd`
- `lttd`
- `drp`

其余字段当前保留为透传元信息，用于：

- 站点核对
- 调试排查
- 后续分类扩展

## 7. 前端内部整理规范

前端将两份原始接口统一整理为内部直接序列，规则固定为：

1. 从 `data` 读取全部时间片
2. 将时间键标准化为 `ISO 8601 +08:00`
3. 按时间升序排序
4. 从全部时间片提取 `stcd`
5. 对 `stcd` 去重并升序排序
6. 建立稳定 `stationIds`
7. 按 `frameIndex * stationCount + stationIndex` 写入 row-major `Float32Array`
8. 将 `drp` 作为唯一数值来源写入矩阵

说明：

- 同一时间片内站点数组顺序不可靠
- 两类产品都必须按统一 `stcd` 顺序重建矩阵
- 内部缺测位置统一填 `NaN`

---

## 8. 校验规范

### 8.1 单接口校验

前端在加载单份原始接口时必须校验：

1. `code == "0"`
2. `data` 非空
3. 每个时间片至少包含一个站点
4. 同一产品内各时间片站点集合完全一致

### 8.2 双接口联合校验

前端在同时加载两份接口后必须校验：

1. `5分钟分时` 时间步长为 `5分钟`
2. `1小时累计` 时间步长为 `60分钟`
3. 两份接口整理后的 `stationIds` 集合完全一致

任一失败时：

- 禁止启动计算
- 返回 `PACKAGE_VALIDATION_FAILED`

---

## 9. 业务语义

| 接口 | `drp` 语义 |
| --- | --- |
| `5分钟分时` | 当前 `5分钟` 分时雨量 |
| `1小时累计` | 当前 `1小时步长` 的起报累计雨量 |

约定：

- 前端不再从 `5分钟` 数据推导累计产品
- `1小时累计` 以接口原始返回为准
