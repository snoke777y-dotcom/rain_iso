# 雨量等值面最小任务定义

## 1. 文档目的

本文用于把“本轮非编码工作做到什么程度算完成”明确下来，作为当前阶段的收口定义。

本文不是实现方案全文，也不是开发任务清单，而是对以下问题给出最终答案：

- 这次要做的最小范围是什么

- 哪些产物必须具备
- 什么条件满足后，当前这轮“需求与设计阶段”才算完成

## 2. 最小任务目标

本轮最小任务目标定义为：

在**不进行任何编码**的前提下，完成雨量等值面浏览器端方案的最小可开发定义，使后续开发人员能够在不反复追问业务口径的情况下直接进入实现阶段。

换句话说，本轮完成标准不是“代码可运行”，而是：

## 4. 已冻结的核心口径

### 4.1 产品口径

本轮已冻结两类产品：

1. `5分钟分时雨量等值面`
2. `起报累计雨量等值面`

说明：

- 分时产品按 `5分钟` 帧生成
- 累计产品按 `1小时` 步长输出
- 累计语义为 `起报累计`

### 4.2 空间口径

- 采用预生成 `1km x 1km` 规则网格
- 使用北京市行政边界 `mask`
- 运行时不临时建网

### 4.3 计算位置口径

- 计算在浏览器客户端完成
- 服务端不承担成图计算
- 必须先整包下载，再开始计算

### 4.4 站点参与口径

- 当前帧先读取全部站点
- `P = 0` 的站点不参与计算
- 全部有效站点参与成图

### 4.5 锚点口径

锚点分为两类：

- `固定锚点`：`121个重点堡垒站 + 5个通州站 + 跨界沟道站`
- `动态锚点`：当前帧 `Top30`

说明：

- 当前口径不是“固定 126 锚点”
- 当前口径是“固定锚点 + 动态锚点”

### 4.6 同格裁决口径

- 同一网格有锚点时，锚点优先
- 锚点冲突按优先级裁决
- 同优先级取最大值
- 无锚点但多普通站时，取“数值最接近最近活动锚点值”的普通站

### 4.7 场生成口径

- 全部有效站点先直接落格
- 先形成“部分已知格场”
- 再对空白格做连续传播补全
- 传播规则以“与相邻已确定网格值最接近”为主
- 最后做受约束平滑

### 4.8 约束层级口径

当前已冻结三层约束：

1. `硬锚点`
2. `软观测点`
3. `待补全格`

其中：

- 硬锚点值不可改写
- 软观测点允许有限微调
- 待补全格自由度最大

### 4.9 渲染口径

- 页面只展示连续色带等值面
- 同时必须保留网格雨量值
- 非雨区透明
- `0 < P < 0.1mm` 不渲染

### 4.10 图例口径

- 使用全局固定图例
- `5分钟` 产品使用 5 分钟图例
- `>=20mm` 新增深紫色
- 累计产品使用 `24小时` 图例体系
