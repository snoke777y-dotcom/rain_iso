## 3. 当前结论

- 浏览器端 CPU 链路可继续作为数值基线。
- 后端选择协议已支持 `webgpu / webgl2 / cpu`，任务启动后不再中途切换。
- GPU 相关模块当前先复用 CPU 算法以保证一致性，尚未实现真实 shader 加速。
- 图例和透明区规则已通过回归测试。
- LRU 帧缓存和数组复用已具备最小可运行能力。
- 浏览器时间轴已支持 `1x / 2x / 3x` 播放，拖动和切帧响应已从固定间隔调度切到按需调度。
- 结果画布现已按容器同比例缩放，默认保持单页显示，不再依赖页面滚动。
- 真实联调后，`softObs` 格点已改为“不允许低于原始观测值”。
- 栅格渲染阶段已改为局部窗口保峰值采样，避免导出图面对局部最大量级再做均值稀释。
- 图面上此前看到的规则“点点”，本质上是超采样后落在原始网格中心的像素痕迹，不是额外站点或新生成观测点。
- 当前联调口径已改为隐藏所有原始网格中心像素，只保留色带轮廓，不再把每个网格中心单独钉出来。
- 最近一轮瓶颈排查表明，首帧慢的主因不在单帧计算，而在 `render-grid-layer` 渲染阶段。

## 4. 参数状态

当前参数已更新为当前联调口径，不是最终业务校准结果：

- 5 分钟 rain_mask 半径：`3 ~ 10`
- 累计 rain_mask 半径：`6 ~ 20`
- 硬锚点扩展步数：`1`
- 传播权重：连续性 `0.7`，锚点参考 `0.3`
- 平滑轮数：`12`
- 软观测最大上浮量：`20mm`
- 软观测下限：`不得低于原始观测值`
- 栅格导出超采样：`pixelScale = 4`
- 局部采样规则：窗口内取最大量级
- 网格中心像素：全部隐藏
- 中心像素替换规则：可渲染雨格中心改为跟随相邻非中心像素，不可渲染格中心保持透明
- 默认平滑参数代码入口：`app/infrastructure/rain_iso/cpu/smooth-params.ts`

## 6. 2026-06-30 性能优化回写

本轮针对“时间轴不流畅、首帧点击后延迟明显、总耗时被错误优化拖慢”的问题，已补以下收口：

- 时间轴播放改为动态 `setTimeout` 调度，并增加 `1x / 2x / 3x` 倍速
- 画布增加按容器同比例缩放，单页展示不再出现滚动条
- 渲染链路拆成独立 render worker，并增加空闲时预热
- 单任务内复用 `stationMetaById / validStationIds / gridIdByStationId`，去掉每帧重复构建
- `render-grid-layer` 内部把格点字符串 `Map` 查找替换为定长数组查表，同时去掉主像素循环里的重复图例分派

本轮已验证的数据：

- 同一份离线样例下，首帧计算约 `40.5ms`
- 同一份离线样例下，首帧渲染由约 `3820.1ms` 降到约 `677.2ms`
- 完整 `195` 帧任务总耗时约 `10.48s`

说明：

- 上述结果来自仓库内样例的离线脚本复测，可证明“总计算时长未被这轮优化拖慢”
- 由于当前自动化环境无法直接读取 localhost 浏览器可见上屏时间，这里暂不把“用户可见首帧时间”写死为最终结论
- 浏览器实页还需要继续用本地页面再复测一次首帧上屏时间
